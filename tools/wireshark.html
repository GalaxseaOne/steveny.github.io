Wireshark
Analyseur de protocole réseau — Capture et inspection du trafic en temps réel
Réseau Analyse PCAP
⚠️
Capturez uniquement le trafic sur des réseaux dont vous êtes propriétaire ou autorisé à analyser. L'interception non autorisée est illégale.

📦 Installation & lancement

Installation
# Debian / Ubuntu / Kali
sudo apt install wireshark tshark

# Permettre à l'utilisateur de capturer sans sudo
sudo usermod -aG wireshark $USER

# Lancement CLI en temps réel
wireshark -i eth0 -k

# Capture et sauvegarde
wireshark -i eth0 -k -w capture.pcap

# Ouvrir un fichier PCAP existant
wireshark capture.pcap

🖥️ tshark — Wireshark en ligne de commande

tshark CLI
# Lister les interfaces disponibles
tshark -D

# Capturer sur eth0
tshark -i eth0

# Capturer avec filtre BPF
tshark -i eth0 -f "tcp port 80"

# Sauvegarder dans un fichier
tshark -i eth0 -w output.pcap

# Lire et filtrer un fichier PCAP
tshark -r capture.pcap -Y "http"

# Extraire les champs spécifiques
tshark -r capture.pcap -T fields -e ip.src -e ip.dst -e http.request.uri

# Statistiques protocoles
tshark -r capture.pcap -q -z io,phs

# Top conversations
tshark -r capture.pcap -q -z conv,tcp

# Extraire les credentials HTTP Basic Auth
tshark -r capture.pcap -Y "http.authorization" -T fields -e http.authorization

# Suivre un flux TCP spécifique
tshark -r capture.pcap -z follow,tcp,ascii,0

À saisir dans la barre de filtre de Wireshark. Cliquez sur une ligne pour copier.

ip.src == 192.168.1.1 and tcp.port == 80
http or dns

Display Filters — HTTP / Web
# Requêtes HTTP uniquement
http.request

# Réponses HTTP
http.response

# Par méthode
http.request.method == "POST"
http.request.method == "GET"

# Par code de réponse
http.response.code == 200
http.response.code == 401
http.response.code >= 400

# Recherche dans l'URI
http.request.uri contains "login"
http.request.uri contains "admin"

# Credentials en clair
http.authbasic
http contains "password"
http contains "passwd"

# Cookies
http.cookie
http.set_cookie
Display Filters — DNS, FTP, SMB
# DNS queries
dns.qry.name contains "target.com"
dns.qry.type == 1    # A records
dns.qry.type == 28   # AAAA records
dns.flags.response == 0   # Requêtes seulement

# FTP credentials
ftp.request.command == "USER"
ftp.request.command == "PASS"

# SMB (recherche de shares)
smb2
smb2.cmd == 3    # Tree Connect

# ARP scanning / spoofing
arp
arp.opcode == 1   # ARP requests

# Paquets TCP avec flag RST
tcp.flags.reset == 1

# SYN scan detection
tcp.flags.syn == 1 and tcp.flags.ack == 0

🔧 Fonctions avancées

💡
Follow TCP Stream : Clic droit sur un paquet → "Follow" → "TCP Stream" pour reconstituer une session complète en clair.
💡
Export Objects : File → Export Objects → HTTP pour extraire les fichiers transférés (images, exécutables, documents).
ℹ️
Déchiffrement SSL/TLS : Avec la clé privée ou le fichier SSLKEYLOGFILE, Wireshark peut déchiffrer HTTPS. Edit → Preferences → Protocols → TLS.
ℹ️
Statistics → IO Graph : Visualiser le trafic dans le temps, détecter les pics suspects ou les exfiltrations de données.
Commandes utiles — extraction de données
# Extraire les credentials FTP d'un PCAP
tshark -r capture.pcap -Y "ftp" -T fields -e ftp.request.command -e ftp.request.arg

# Extraire les URLs visitées
tshark -r capture.pcap -Y "http.request" -T fields -e http.host -e http.request.uri

# Extraire les hashes NTLMv2 (pour cracking)
tshark -r capture.pcap -Y "ntlmssp.auth"

# Convertir PCAP en texte lisible
tcpdump -r capture.pcap -A | grep -i "password\|user\|login"
