Nikto
Scanner de vulnérabilités web — Fichiers dangereux, mauvaises configs, versions obsolètes
Web Scanner CVE

📦 Installation

Installation
# Kali / Debian / Ubuntu
sudo apt install nikto

# Depuis GitHub
git clone https://github.com/sullo/nikto.git
cd nikto/program && perl nikto.pl -h

# Mettre à jour les plugins
nikto -update

📋 Cheatsheet Nikto

Scans de base

Commandes essentielles
# Scan de base
nikto -h http://target.com

# Sur HTTPS
nikto -h https://target.com -ssl

# Port personnalisé
nikto -h 192.168.1.1 -p 8080

# Scan HTTPS sur port 443
nikto -h 192.168.1.1 -p 443 -ssl

# Avec authentification basique
nikto -h http://target.com -id admin:password

# Avec cookie de session
nikto -h http://target.com -cookies "PHPSESSID=abc123"

# Scan de plusieurs hôtes (fichier)
nikto -h targets.txt

Sortie et rapports

Formats de sortie
# Rapport HTML
nikto -h http://target.com -o rapport.html -Format htm

# Rapport XML
nikto -h http://target.com -o rapport.xml -Format xml

# Rapport CSV
nikto -h http://target.com -o rapport.csv -Format csv

# Texte simple
nikto -h http://target.com -o rapport.txt

Options avancées

Tuning et évasion
# Via proxy Burp Suite
nikto -h http://target.com -useproxy http://127.0.0.1:8080

# Tuning (tests spécifiques)
nikto -h http://target.com -Tuning 1   # Fichiers intéressants
nikto -h http://target.com -Tuning 2   # Mauvaises configs
nikto -h http://target.com -Tuning 3   # Divulgation d'infos
nikto -h http://target.com -Tuning 4   # Injection (XSS/HTML)
nikto -h http://target.com -Tuning 9   # SQL injection
nikto -h http://target.com -Tuning x   # Tout sauf DoS

# User-Agent personnalisé
nikto -h http://target.com -useragent "Mozilla/5.0"

# Ignorer les redirections
nikto -h http://target.com -nofollow

# Timeout personnalisé
nikto -h http://target.com -timeout 10

# Combinaison complète
nikto -h http://target.com -Tuning x -o scan.html -Format htm -ssl -useproxy http://127.0.0.1:8080

🔍 Ce que Nikto détecte

Types de vulnérabilités
✓ Fichiers et programmes potentiellement dangereux
✓ Versions de serveur obsolètes (Apache, Nginx, IIS)
✓ Configurations serveur incorrectes
✓ Headers de sécurité manquants (CSP, HSTS, X-Frame-Options)
✓ Répertoires sensibles (admin/, backup/, config/)
✓ Fichiers de sauvegarde (.bak, ~, .old)
✓ Pages d'erreur avec informations de debug
✓ Formulaires sans protection CSRF
✓ Injection XSS basique
✓ Injection SQL basique
✓ Divulgation d'informations via headers HTTP
✓ Méthodes HTTP dangereuses (PUT, DELETE, TRACE)
✓ Fichiers robots.txt et sitemap.xml
✓ Certificats SSL/TLS expirés ou mal configurés
✓ PHPInfo() accessible
✓ Répertoires .git, .svn exposés
💡
Conseil : Nikto est bruyant (non-furtif). Il génère beaucoup de trafic et sera détecté par la plupart des IDS/WAF. Utilisez-le en dernier recours ou après avoir évalué les risques de détection.
